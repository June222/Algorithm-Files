### 10989 - 수 정렬하기 3

# 사용된 알고리즘 :
	1. 정렬

# 해결 목표 :
	1. 10,000,000개의 숫자가 주어진다. 이 숫자를 정렬하여라.
	2. 메모리 제한이 있다. 8MB이다. 

# 해결해야 할 부분 :
	1. 메모리 제한을 꼭 보자.
	-> 대부분의 경우에 문제는 주로 시간복잡도에 초점이 맞춰져있다.
	-> 그런데 종종 공간복잡도에 초점이 맞춰져있는 문제도 있다.
	-> 8MB는 무엇을 의미할까?


	2. 8MB에 얼마나 많이 저장될 수 있는가
	-> int 자료형 정수는 4(2^2)byte를 필요로한다.
	-> 참고로 1byte는 8bit를 의미한다.
	-> 1024(2^10)byte는 1KB이고
	-> 1024(2^10)KB는 1MB이다.
	-> 8MB는 8개의 1MB이므로 2^3MB이다.
	-> 따라서 8MB는 2^23byte이고, int 하나에 2^2byte가 필요하므로
	-> 2^21개의 숫자가 저장가능하다. (약, 200만개)

	
	3. 10,000,000는 몇 MB?
	-> 10MB이다. 따라서 저장한다면 메모리 초과가 날 것이다.
	

	4. 해결방안
	-> 숫자가 1만개 밖에 없다는 것에 주목하자.
	-> 숫자가 몇 개가 있는지 안다면 정렬할 필요없지 않을까?
	-> count한 뒤에 작은 수부터 수만큼 출력한다.
