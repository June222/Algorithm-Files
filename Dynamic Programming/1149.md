### 1149 - RGB거리

# 사용된 알고리즘 :
	1. Dynamic Programming

# 해결 목표 :
	1. N개의 집을 빨강, 초록, 파랑으로 색칠하는데 각 집마다, 각 색마다 다른 비용이 든다.
	2. 1번 집의 색은 2번 집의 색과 달라야 한다.
	3. N번째 집의 색은 N-1번째 집의 색과 달라야 한다.
	4. i번째(2 <= i <= N-1) 집의 색은 i-1, i+1번째 집의 색과 달라야 한다.
	5. 이때, 최소비용을 구하시오.

# 해결해야 할 부분 :
	1. 시간 제한 0.5초
	-> 가장 중요한 부분인데 항상 정확히 계산을 하고 들어가지 않는다.
	-> 처음에는 DFS로 풀 생각을 했다. 비슷하다고 생각이 드는 문제가 있었기 때문이다.
	-> 하지만, 모든 계산을 끝내면 2^1000의 시간이 든다...
	
	-> 따라서 다른 알고리즘이 필요하다.


	2. 계산을 줄이는 Dynamic Programming
	-> 생각은 했지만 어떻게 구현해야하는지 생각하지 못했다.
	-> 생각을 할 땐 항상 작게 나누어 먼저 생각해보자.

	-> 1) 모든 집에 3가지 색에 대한 경우를 포함해야한다.
	-> 2) 2개의 집만이 있다고 생각해보자.
	-> 3) 2번째 집에 빨간색을 칠하는 경우는 언제 최소인가?
	-> 4) 우선 1번째 집은 빨간색을 칠할 수 없다. 그럼 2가지만 고려한다.
	-> 5) 1번째 집의 G or B 중에 제일 싼 값과 빨간색이 칠해지면 최소가 된다.
	-> 6) 같은 방식으로 2번째 집에 나머지 두 색을 칠하고 최소값을 저장한다.
	-> 7) 3번째 집은 1번째 집을 고려해야할까?
	-> 8) 이미 첫번째 집의 색이 고려된 최소값이 2번째에 저장되어 있다. 따라서 i번째 집은 i-1번째 집과만 비교하면 된다.
	-> 9) 시간복잡도 : O(N * 6)
