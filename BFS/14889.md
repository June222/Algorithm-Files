### 14889 - 스타트와 링크

# 사용된 알고리즘 :
	1. 백트래킹

# 해결 목표 :
	1. Promising에 대한 부분
	2. 백트래킹을 시행할 때, 매 노드마다의 초기화
	3. 점수 체크를 언제 할 것 인가?

# 해결해야 할 부분 :
	1. Promising은 여기서 따로 구현하지 않음
	-> 왜냐하면 (1,2), (3,4)와 (3,4), (1,2)가 똑같기 때문에 한번만 결정하면 됨
	-> 팀에 있나 없나 계산할 필요가 없이 그냥 for문 시행

	2. 2. 백트래킹 실행 시에 백트래킹 안에서 for문으로 시행하는 점을 이용
	-> for문 안에서 조건을 적용하고 함수 종료 이후에 다시 초기화 하는 것으로 
	초기화를 매 노드마다의 초기화를 진행할 수 있음.  

	3. Promising에 대한 구현이 필요 없음.
	-> 시간을 아끼려고 매 노드마다의 검사를 진행하였으나 오히려 모든 배열에 대한 
	점수 계산을 시간을 더욱 느리게 만들 수 있음(빠른 점수 계산 방법을 연구할 필요)
	-> 따라서 cnt가 종료에 도달하였을 때, 계산하는 것이 여러 번의 계산을 막을 수 
	있음.
