### 15686 - 치킨 배달

# 사용된 알고리즘 :
	1. BackTracking (Brute Force)

# 해결 목표 :
	1. 주어진 맵에서 치킨집을 정해진 개수만큼만 활성화 시킬 때
	2. 집에서 가장 가까운 치킨집 거리의 합들 중 최소가 되는 거리를 출력하라.

# 해결해야 할 부분 :
	1. Greedy 알고리즘으로 풀 수 없는 반례가 있다.
	
	ex)	5 4			
		2 1 0 1 2 
		0 0 0 0 0 
		0 0 2 0 0 
		0 0 0 0 0
		2 1 2 1 2 

	-> 이런 상황이라면 치킨집을 어떻게 골라야할까?
	-> 또, 어떻게 조건을 걸어야할까?
	-> 만약 집에서(1) 가장 가까운 치킨집(2)를 고른 다는 조건을 걸면 다음 예시를 제대로 찾지 못한다.

	ex) 5 1
		2 1 0 1 2 
		0 0 0 0 0 
		0 0 2 0 0 
		0 0 0 0 0
		2 1 2 1 2 

	-> 따라서 시간복잡도를 계산한 이후에 모두 다 해보는 BackTracking 알고리즘을 사용한다.
	-> 시간복잡도 : 
		1) 집: 100개, 가게: 13개
		2) 가장 많이 고르는 경우: 13C6 = 약 1700개
		3) 약 17만개.
		4) 하나의 자료당 500번이 넘는 연산을 하지 않는 경우라면 통과됨.

	2. BackTracking 알고리즘 사용 시에 주의할 점 (반복됨)
	-> BackTracking은 재귀로 구현을 한다.
	-> 이때, 함수에 사용되는 인자를 정확하게 써야한다.
